Supporting Programming Learning in Hybrid Classrooms

Research project exploring how students collaborate, communicate, and learn programming in hybrid settings, and how EdTech tools can better support these workflows.

Overview

Understanding low adoption of a programming support tool

A coding-support chatbot developed by the lab had less than 1% usage among students. This study aimed to understand student learning behaviours and identify gaps between existing tools and actual needs.

Problem

Students rely on fragmented tools for collaboration

User challenges

  • Students used multiple tools (WhatsApp, screenshots, Jupyter, email) to collaborate.
  • Sharing code and explaining problems was inefficient.
  • Group work was harder in hybrid settings with limited coordination.

Research question

How can EdTech tools better support communication, collaboration, and learning for programming students in hybrid environments?

Method

Qualitative user research approach

The study combined interviews, participatory workshops, and thematic analysis to understand student behaviour and needs.

Findings

Key insights from student behaviour

Main observations

  • Low awareness of the chatbot.
  • Heavy reliance on informal communication tools.
  • Lack of integrated collaboration platform.
  • Difficulty sharing specific parts of code.

Personas

Nina: Beginner, needs guidance and simple sharing tools.

Elina: More confident, needs efficient collaboration and feedback.

Solutions

Design recommendations

Communication

  • Voice and video calls
  • Chat-based support

Collaboration

  • Real-time code sharing
  • Commenting and highlighting

Control

  • Selective code sharing
  • User roles and permissions
Conclusion

Key takeaways

  • Students need integrated tools, not standalone features.
  • Flexibility and control improve collaboration.
  • Design should align with real user workflows.
